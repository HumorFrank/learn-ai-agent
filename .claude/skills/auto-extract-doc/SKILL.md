# 文档自动提取与分类
自动识别并将 Markdown (.md) 文件提取到一个独立的文档目录中，根据内容自动分类到不同的子目录。

## 工作流程
1. 监控项目中的 .md 文件
2. 当 .md 文件被修改或新增时，自动触发提取流程
3. 读取 .md 文件内容，分析其主题和关键词
4. 根据分析结果将 .md 文件复制到对应的文档目录中
5. 维护一个索引文件，记录每个文档的来源路径、提取时间和分类标签

## 分类规则
- 技术文档：包含技术相关内容的 .md 文件，如教程、规范、技术总结等，提取到 `docs/technical/` 目录
- 面试文档：包含面试相关内容的 .md 文件，如面经、面试题总结等，提取到 `docs/interview/` 目录
- 项目文档：包含项目相关内容的 .md 文件，如项目说明、使用指南、开发文档等，提取到 `docs/projects/` 目录
- ai文档：包含 AI 相关内容的 .md 文件，如 AI 模型介绍、使用指南、AI 项目文档等，提取到 `docs/ai/` 目录
- 其他文档：不符合上述分类的 .md 文件，提取到 `docs/other/` 目录

## 索引文件

- 索引文件格式：`docs/index.json`
- 每条记录包含以下字段：
  - `sourcePath`: 原始 .md 文件路径
  - `extractedPath`: 提取后的文档路径
  - `extractedTime`: 提取时间戳
  - `category`: 分类标签（technical, interview, projects, ai, other）
  - `keywords`: 从文档内容提取的关键词列表
  - `summary`: 文档内容的简要摘要（可选）
- 索引文件示例：
```json
[
  {
    "sourcePath": "src/面经/前端面试题总结.md",
    "extractedPath": "docs/interview/前端面试题总结.md",
    "extractedTime": "2024-06-01T12:00:00Z",
    "category": "interview",
    "keywords": ["前端", "面试题", "总结"],
    "summary": "总结了常见的前端面试题，包括 JavaScript、CSS、HTML 等方面的内容。"
  },
  {
    "sourcePath": "src/技术文档/React 教程.md",
    "extractedPath": "docs/technical/React 教程.md",
    "extractedTime": "2024-06-01T12:30:00Z",
    "category": "technical",
    "keywords": ["React", "教程", "技术文档"],
    "summary": "提供了 React 的基础教程，包括组件、状态管理和生命周期等内容。"
  },
  {
    "sourcePath": "src/项目文档/项目说明.md",
    "extractedPath": "docs/projects/项目说明.md",
    "extractedTime": "2024-06-01T13:00:00Z",
    "category": "projects",
    "keywords": ["项目", "说明", "文档"],
    "summary": "介绍了项目的背景、目标和主要功能。"
  }
]
```
## 使用场景

- 当用户提及文档提取、Markdown 分类、文档归档时使用
- 批量提取项目中的 .md 文件，自动分类并维护索引文件，方便后续查询和管理
- 定期扫描项目中的 .md 文件，确保新添加或修改的文档能够及时被提取和分类
- 通过索引文件快速查找特定主题的文档，提升文档管理效率

## 注意事项

- 确保提取流程的性能和稳定性，避免对项目开发造成过多干扰
- 分类规则可以根据实际项目需求进行调整和优化，确保分类的准确性和实用性
- 定期清理和维护提取后的文档目录，避免冗余和过时的文档占用空间
- 索引文件应保持最新，确保记录的准确性和完整性，方便后续查询和管理
- 提取流程应具备错误处理机制，确保在提取过程中出现问题时能够及时反馈并进行修复
- 可以考虑添加文档版本控制功能，记录每次提取的文档版本和修改历史，方便追踪文档的演变过程
- 可以结合 AI 技术进行文档内容分析和分类，提高分类的准确性和智能化水平
- 可以提供一个用户界面，方便用户查看和管理提取后的文档，以及查询索引文件中的记录
- 根据实际项目需求调整和优化分类规则，确保分类的准确性和实用性，满足不同类型文档的管理需求

## 强制遵守规则
- 归档时需要排除一些不需要提取的 .md 文件，如 README.md、CHANGELOG.md 等大写英文名文档，避免干扰分类和索引的准确性
- 归档是将 .md 文件移动原文件到对应的文档目录中，而不是复制，方便开发和版本控制
- 白名单目录
  - .claude/ 目录下的文档不予提取，避免干扰技能的正常运行和管理
  - 其他特定目录（如 node_modules/、dist/ 等）下的 .md 文件不予提取，避免干扰项目的正常开发和构建流程
- 只能提取 .md 文件，其他格式的文档不予提取
- 提取后的文档必须按照分类规则存放在对应的目录中
- 索引文件必须保持最新，记录每次提取的文档信息

# Markdown 格式化与结构化整理

参照 [markdown.com.cn](https://markdown.com.cn/) 教程规范，对 Markdown 文档进行格式化与结构化整理。

## 工作流程

1. 读取目标 .md 文件
2. 按以下规则逐项检查并修正
3. 将格式化后的内容写回文件
4. 报告所做的修改

## 格式化规则

### 1. 标题 (Headings)

- 使用 ATX 风格：`#` 后必须有且仅有一个空格，再跟标题文本
  - 正确: `## 标题文本`
  - 错误: `##标题文本`、`##  标题文本`（多余空格）
- 标题前后各留一个空行（文档开头的标题前不需要）
- 文档只有一个 H1 (`#`) 标题
- 标题层级不跳级（H1 → H2 → H3，不出现 H1 直接到 H3）
- 标题末尾不要有多余的 `#` 符号
- 不使用 Setext 风格（下划线）标题，统一用 ATX 风格

### 2. 段落与换行 (Paragraphs & Line Breaks)

- 段落之间用空行分隔（连续文本属于同一段落）
- 段落首行不要用空格或 Tab 缩进
- 段落内换行不使用 `<br>` 标签，除非确实需要（如地址、诗歌）
- 移除行尾多余空格

### 3. 强调 (Emphasis)

- 粗体使用 `**文本**`，斜体使用 `*文本*`
- 粗体+斜体使用 `***文本***`
- 标记符内侧不要有多余空格
  - 正确: `**粗体**`
  - 错误: `** 粗体 **`
- 优先使用 `*` 而非 `_` 作为强调标记

### 4. 列表 (Lists)

- 无序列表统一使用 `-` 作为标记（后跟一个空格）
  - 不使用 `*` 或 `+` 混搭
- 有序列表使用 `1.` 起始，数字不必连续但必须从 1 开始
- 嵌套列表缩进 4 个空格（或 1 个 Tab，全文档统一）
- 列表前后各留一个空行
- 列表项如果是完整句子，首字母大写并加句号；如果是词组，不加句号
- 列表标记符（`-` 或 `1.`）后跟一个空格再跟内容

### 5. 任务列表 (Task Lists)

- 格式：`- [ ]` 未完成，`- [x]` 已完成
- `[ ]` / `[x]` 两侧各留一个空格：`- [ ] 任务描述`
- 任务描述与复选框之间有一个空格

### 6. 代码 (Code)

- 行内代码使用单个反引号包裹：`` `代码` ``
- 代码内容包含反引号时，使用双反引号包裹：`` `` `代码` ``
- 多行代码块使用围栏式（Fenced Code Blocks）：` ``` `
- 围栏代码块必须标注语言标识符：
  ```markdown
  ```javascript
  const a = 1;
  ```
  ```
- 代码块前后各留一个空行
- 不使用 4 空格缩进方式创建代码块（统一使用围栏式）

### 7. 链接与图片 (Links & Images)

- 链接格式：`[显示文本](URL "可选title")`
- 图片格式：`![替代文本](URL "可选title")`
- 图片必须有替代文本（alt text）
- URL 中有空格时，使用 `%20` 替代
  - 正确: `[链接](https://example.com/my%20page)`
  - 错误: `[链接](https://example.com/my page)`
- 裸 URL 用尖括号包裹：`<https://example.com>`
- 引用式链接（Reference-Style）标签不区分大小写
- 引用式链接定义放在文档末尾

### 8. 引用 (Blockquotes)

- 使用 `>` 后跟一个空格
- 多段落引用：每个段落开头加 `>`
- 嵌套引用：使用 `>>`
- 引用前后各留一个空行
- 引用内部可以包含其他 Markdown 元素（标题、列表、代码等）

### 9. 表格 (Tables)

- 表头与内容用 `---` 分隔线分开
- 对齐标记：左对齐 `:---`，居中 `:---:`，右对齐 `---:`
- 每列至少 3 个短横线
- 管道符 `|` 对齐（非强制，但全文档统一风格）
- 表格前后各留一个空行

### 10. 分割线 (Horizontal Rules)

- 使用 `---`（三个短横线）
- 分割线前后各留一个空行
- 全文档统一使用一种风格

### 11. 删除线 (Strikethrough)

- 使用 `~~文本~~`
- 标记符内侧不要有多余空格

### 12. 脚注 (Footnotes)

- 脚注引用：`[^1]` 或 `[^标签名]`
- 脚注定义放在文档末尾

### 13. 通用清理规则

- 文件末尾有且仅有一个空行
- 移除行尾多余空格（trailing whitespace）
- Tab 缩进统一转换为 2 空格（与项目代码风格保持一致）
- 中文与英文、数字之间加空格（如：`使用 React 18`）
- 中文与行内代码之间加空格（如：`使用 `ref` 方法`）
- 连续的空行压缩为单个空行（代码块内的空行除外）

### 14. 文档结构化规则

- 确保文档有清晰的标题层级结构
- H1 标题反映文档主题
- 文档超过 50 行时，在 H1 之后添加目录（Table of Contents）
- 同类内容归入同一章节，避免散落各处的零散段落
- 链接引用定义统一放在文档末尾（脚注区）
- 文章末尾的参考链接统一整理为 `## 参考资料` 章节

## 格式化检查清单

每次格式化时，按以下顺序检查：

1. 标题格式（空格、空行、层级）
2. 段落间距（多余空行、行尾空格）
3. 列表标记统一（`-` 无序、`1.` 有序）
4. 代码块语言标识
5. 链接格式和裸 URL
6. 表格对齐
7. 中英文空格
8. 文件末尾空行
9. 文档结构（目录、层级合理性）
